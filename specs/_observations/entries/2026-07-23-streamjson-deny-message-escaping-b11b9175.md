- 2026-07-23 [planwright] fleet-streamjson.sh cmd_answer builds the deny body by escaping only backslash and double-quote, while its sibling json_escape_file also escapes tab and carriage return and strips remaining C0 and DEL bytes. An operator deny message containing a literal tab or CR therefore emits a raw control byte inside a JSON string, which is invalid per RFC 8259 and can make the worker reject or mis-parse the control_response. Fix: route the deny message through the same escaping json_escape_file applies. Pre-existing on main (Task 4 code); surfaced by a panel-review gemini pass during Task 7.
