- 2026-07-22 [planwright] worker-settings.json wired via `claude --settings <profile>` fails at runtime: its PreToolUse hook references ${CLAUDE_PLUGIN_ROOT}, which Claude Code resolves only for hooks defined in a plugin hooks/hooks.json, never for hooks in a settings file ("Hook command references ${CLAUDE_PLUGIN_ROOT} but the hook is not associated with a plugin"). The profile doc claims the reference "resolves under a marketplace install" — true only for plugin-registered hooks, and the profile is deliberately NOT plugin-registered (REQ-C1.2). Any --settings delivery path therefore ships a dead guard hook. Fix: profile needs a documented literal-path substitution step (or a generator) when delivered via --settings, alongside the already-documented literal-path Bash allow entry.
