- 2026-07-23 [planwright] fleet-status.sh joins oracle sessions to workers by the sid in each worker`s streamjson/<w>/session file (hand-corruptible per the posture) and drops any oracle row whose sid a worker claims. A hostile/buggy worker writing another session`s id (e.g. the operator`s interactive session) into its own session file suppresses that oracle session from the unjoined-sessions list and misattributes its busy/waiting status to the worker row — a display-integrity gap the Task 8 dashboard inherits. Mitigation ideas: cross-check the joined session`s cwd against the worker`s worktree, or flag a sid claimed by >1 worker.
