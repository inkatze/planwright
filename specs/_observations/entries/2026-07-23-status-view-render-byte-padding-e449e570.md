- 2026-07-23 [planwright] fleet-status.sh render pads columns with printf '%-20s ...', which counts BYTES, while the echo-safety sanitizer strips only C0/DEL/C1 (0x00-0x1F, 0x7F, 0x80-0x9F) — so multibyte UTF-8 in a session name or scope survives and shifts that row's columns. CLI-render-only and cosmetic: nothing is hidden or truncated (columns overflow rather than clip), and the tab-separated merge stream (Task 8's contract) is unaffected. Reviewed 2026-07-23 during copilot-review on PR #315 and deliberately left as-is: a wcwidth-aware fix in POSIX sh is disproportionate to a cosmetic gain, and stripping non-ASCII would mangle legitimate worker names. Task 8's HTML/browser dashboard sidesteps byte-padding natively (the browser lays out text), so this constraint belongs to the CLI renderer alone and is not a reason for the dashboard to sanitize further.
