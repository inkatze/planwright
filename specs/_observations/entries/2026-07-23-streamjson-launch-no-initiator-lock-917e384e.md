- 2026-07-23 [planwright] fleet-streamjson.sh cmd_launch has no single-initiator lock, unlike cmd_recover which elects one with mkdir recover.lock. Two concurrent launches for the same worker both mkdir -p the runtime dir, both remove and re-mkfifo in.fifo and out.fifo inside supervise, and both overwrite supervisor.pid and worker.pid, so one supervisor is orphaned writing into a fifo nobody reads and the recorded pids describe only the last writer. Fix: elect a launch initiator with the same atomic mkdir pattern cmd_recover uses, refusing the second caller. Pre-existing on main (Task 4 code); surfaced by a panel-review gemini pass during Task 7.
