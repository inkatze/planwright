- 2026-07-23 [planwright] fleet-streamjson.sh cmd_answer bounds the operator deny message with head -c 512, a byte bound, so a message longer than 512 bytes whose 512th byte falls inside a multibyte UTF-8 sequence is truncated mid-character and emits a malformed UTF-8 fragment into the control_response JSON. The same file deliberately preserves UTF-8 elsewhere (json_escape_file keeps bytes at or above 0x80 precisely so accents, CJK and emoji reach the worker intact), so the byte-truncation is inconsistent with that stated intent. Fix: trim back to a character boundary after applying the byte bound. Low reachability (needs a long multibyte message). Pre-existing on main (Task 4 code); surfaced by a panel-review gemini pass during Task 7.
