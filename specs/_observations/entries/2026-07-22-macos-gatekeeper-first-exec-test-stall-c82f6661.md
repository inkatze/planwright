- 2026-07-22 [planwright] Tests that copy scripts into a temp dir and exec them directly stall for tens of seconds per first-exec on macOS (Gatekeeper/XProtect first-run assessment; observed 10-57s per invocation while implementing format-grammar Task 1). Invoking the copy via `sh <script>` reads it as data and avoids the assessment entirely (0.06s). Any suite that fabricates executable fixtures should prefer interpreter invocation; worth a sweep of existing tests that direct-exec fresh temp copies.
