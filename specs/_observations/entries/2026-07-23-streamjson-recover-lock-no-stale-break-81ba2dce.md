- 2026-07-23 [planwright] fleet-streamjson.sh cmd_recover elects a single recovery initiator with mkdir recover.lock and releases it only through an EXIT trap, with no stale-breaking path. A SIGKILL or host reboot mid-recovery leaves the directory behind permanently, so every later recover for that worker is refused with recovery-already-in-progress and the worker can never be recovered without manual filesystem surgery. Its sibling journal_lock already implements an mtime-based 60s stale break, so the asymmetry looks unintended. Fix: give recover.lock the same stale-age break, or record the initiator pid inside the lock and break it when that pid is provably dead via fleet-death-evidence.sh. Pre-existing on main (Task 4 code); surfaced by a panel-review gemini pass during Task 7.
