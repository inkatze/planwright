# Instruction-budget suppression list (prompt-hygiene REQ-B1.3).
#
# scripts/check-instructions.sh reads this file as DATA (never evaluated,
# REQ-B1.9). Blank lines and lines beginning with '#' are ignored. Two forms
# exist; each REQUIRES a recorded reason (a reason-less entry of either form is
# an error, REQ-B1.3 / REQ-B1.8):
#
#   exempt|<path>|<reason>
#       Permanent per-file-floor exemption ONLY. It suppresses a file's per-file
#       floor and NEVER the mandatory-at-start or reachable-closure budgets
#       (REQ-B1.3a). Carries a standing rationale.
#
#   pending-diet|<budget>|<target>|Task <N>|<reason>
#       Transitional allowance. The only form that may cover a mandatory-at-start
#       or reachable-closure offender, and only transiently. Removed by that
#       diet task's own PR, and forbidden at closeout (REQ-B1.3b, REQ-D1.4).
#       <budget> is one of: file | start-load | closure
#       <target>  is a file path (file) or a skill name (start-load | closure)
#
# Seeded below: the per-file offenders the initial audit surfaced, each carrying
# a transitional allowance until its diet task lands (Tasks 5/6/7). The
# start-load allowance for /spec-draft is seeded at Task 3, when manifests first
# make start-load computable — not here, where no manifest yet exists.
pending-diet|file|skills/spec-kickoff/SKILL.md|Task 7|/spec-kickoff SKILL.md is over the per-file SKILL floor; dieted in Task 7.
pending-diet|file|doctrine/spec-format.md|Task 7|spec-format.md is over the per-file doctrine floor; trimmed or permanently exempted in Task 7.

# Start-load allowances seeded at Task 3, when manifests first made mandatory-at-start
# computable (REQ-A1.2, REQ-B1.3b). Each start-load offender the manifest computation
# surfaced is carried transiently until its diet lands; a start-load offender has no
# permanent-exemption escape valve (REQ-B1.3a forbids it), so this is the only form
# that can keep the check green while the diet is pending. No reachable-closure offender
# surfaced (every skill's closure is under the 20,000 error threshold), so no closure
# allowance is seeded. Each is removed by its diet task's own PR (Tasks 5/6/7 shed both
# their per-file and start-load allowances; /spec-draft's start-load diet is Task 7.5).
pending-diet|start-load|spec-kickoff|Task 7|/spec-kickoff mandatory-at-start is over budget (body + run-start doctrine); dieted in Task 7.
pending-diet|start-load|spec-draft|Task 7.5|/spec-draft mandatory-at-start is over budget purely through run-start doctrine (body under the per-file floor, so invisible to Task 2's pre-manifest audit); dieted in Task 7.5 by point-of-use reclassification.
