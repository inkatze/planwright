# Reference material (seed, not part of planwright)

`pair-flow/` is copied verbatim from the dotfiles repo's `specs/pair-flow/`
at commit `1e19687` (2026-06-08), as seed material for planwright's bootstrap
spec.

This is **source to mine, not planwright proper**. It captures the system being
extracted: the four-file bundle, the signed-off kickoff brief, and the v1
retrospective. `research/v1-retrospective.md` Section 4 (personal-preference vs
generalizable) is the in-scope / out-of-scope seed; Section 5 (what v1 did not
validate) is the risk seed.

`dotfiles-claude/` is the **live process source** being extracted, copied from the
dotfiles repo at commit `1e19687`: the global `CLAUDE.md` (the rule "brain"), all
skill command files (`claude/commands/`), all hook scripts (`claude/scripts/`),
`pair-flow.yml`, `settings.json` (hook wiring), and `specs-README.md` (the four-file
format conventions). These are the actual artifacts the extraction operates on, not
descriptions of them.

Curate or delete this directory once `specs/bootstrap/` is drafted and stable.
